Current State

A private-network file server controlled by a daemon.

serve-lib lets users register local files and directories once, then let a background daemon serve them on Tailscale or another private interface. Today an experimental daemon, CLI, HTTP listener, control API, events, timeout scheduler, and rustls-backed TLS/mTLS sockets exist.

User Workflow Preview

Start Daemon

Run the experimental daemon on a private local control API. The serving listeners are opened only when paths are registered. 

target/release/serve-lib --control 127.0.0.1:7878 daemon run

Register Paths

A single daemon-owned port can expose multiple local directories or files under different URL subpaths. 

target/release/serve-lib --control 127.0.0.1:7878 register ./dist --route /app --port 8088 --bind loopback
target/release/serve-lib --control 127.0.0.1:7878 register ./logs --route /logs --port 8088 --bind loopback
target/release/serve-lib --control 127.0.0.1:7878 list

Audit Activity

SQLite events should let users inspect what was registered, who accessed it, and when old records were cleaned up. 

target/release/serve-lib --control 127.0.0.1:7878 events
target/release/serve-lib --control 127.0.0.1:7878 daemon stop

These commands use the experimental CLI. For full human and agent continuation steps, see docs/codelabs.md.

Control Plane

The CLI is short-lived. It builds register/deregister/list requests and sends them to the daemon over a private HTTP JSON control API. Config mutation commands are still pending.

Data Plane

The daemon owns listeners and serves HTTP traffic. Static file planning, path safety, directory listing, and SPA fallback are reusable core behavior.

Event Plane

SQLite records route lifecycle and access events. Retention is TTL based, not max-row-count based. A background cleanup worker exists in the runtime.
